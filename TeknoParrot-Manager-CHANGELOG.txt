  v0.68 BETA  (2026-06-14)
-------------------------------------------------------------------------------

  Bug fixes -- AutoSync crash and ZIP source paths not saved between runs:

  - Fixed a crash at startup of AutoSync: the queue header rendered an inline
    (if ...) expression which PowerShell 5.1 does not support as a cmdlet
    argument. Changed to $(if ...) subexpression syntax.

  - Fixed the main collection ZIP folder and supplementary ZIP folder not being
    remembered between runs. Both paths are prompted inside the main menu loop

  - Fixed the supplementary folder prompt re-appearing every run when the user
    had explicitly skipped it (pressed Enter). The empty response is now saved
    and the prompt condition distinguishes "never asked" ($null) from "user
    said no" ('') so it does not repeat.

  Bug fix -- D (Done) now accepted with an empty queue in game pickers:

  - Pressing D with no games in the queue now proceeds correctly instead of
    blocking with "Queue is empty." D always means "proceed with whatever I
    selected"; an empty queue means zero games are extracted. Use A to

  AutoSync: combined game picker, extraction progress bar, network staging:

  - Game selection for AutoSync now combines the main collection and the
    supplementary library into a single A-Z picker. Supplementary entries

  v0.59 BETA  (2026-06-14)
-------------------------------------------------------------------------------

  GitHub profile set resolution, supplementary dat as priority overrides:

  - The script now fetches the full list of GameProfile XML filenames from the
    teknogods/TeknoParrotUI GitHub repository before every registration run
    (GET /repos/teknogods/TeknoParrotUI/git/trees/master?recursive=1). This
    profile set is used to resolve dat ProfileCodes that do not exactly match
    any local template: a fuzzy (Sorensen-Dice) match against the set is tried
    and, if above the auto-register confidence threshold, the matched template
    name is used instead. This fixes games like BladeArcusFromShining in the
    dat that must map to BladeArcus.xml in GameProfiles. If GitHub is
    unreachable, the script falls back to scanning the local GameProfiles folder.
    Fetched stems are validated [\w]+ before being added to the set; fuzzy
    matches are re-validated before being returned as path components.

  - Supplementary dat entries are now treated as priority overrides for the
    collection dat (alternate versions that replace the collection version on a
    game-by-game basis -- only one version can be installed at a time in
    TeknoParrot). When both dats are loaded, supplementary entries are merged
    over the collection index so the supplementary version is used for any
    game present in both. The "alternate versions" display in the post-
    registration Game Info block is replaced with a "Source: supplementary dat"
    tag for games registered via a supplementary override. The Build-Supplementary*
    family of functions is removed; the supplementary dat is now parsed with
    Build-DatIndexFromZip / Build-DatIndex using a '*Supplementary*_RomVault*.dat'
    entry pattern, producing the same normalizedName -> {ProfileCode, Executable}
    format as the collection index.

  v0.57 BETA  (2026-06-14)
-------------------------------------------------------------------------------

  Eggman dat auto-download and ZIP-based dat reading:

  - The initial setup now offers four dat configuration options: download the
    Eggman collection ZIP directly from GitHub (~145 MB), point to a local
    copy of the ZIP, use separate collection and supplementary dat files, or
    skip. The dat path entered during setup is saved to config.json and shown
    on subsequent runs. Choosing Download queries the GitHub API for the latest
    release asset (Eggmansworld/Datfiles, teknoparrot tag), shows the file name
    and size, asks where to save it (default: Scripts folder), then downloads
    with BITS (shows a progress bar) and falls back to WebClient if BITS is
    unavailable. If GitHub cannot be reached or the download fails, the script
    falls through to a manual path prompt.

  - Both dats inside the ZIP are now read directly via streaming XmlReader
    without extracting the archive. This means no temporary extraction step,
    no disk space overhead, and the 584 MB collection dat is processed in a
    single forward pass that skips rom hash nodes entirely. The ZipArchive is
    held open only for the duration of the read and is disposed immediately
    after, whether the parse succeeds or not.

  - The supplementary dat (44 MB inside the ZIP) can now be optionally indexed
    alongside the collection dat. The supplementary index maps each ProfileCode
    to the list of alternate game names (different regions, revisions, or bonus
    content). When supplementary indexing is enabled, the count of alternate
    entries is shown after loading. The index is held in memory for future use
    by the post-registration display (planned for v0.58).

  - Build-DatIndex (previously DOM-based, unsuitable for 584 MB files) now uses
    streaming XmlReader internally. The public signature is unchanged -- existing
    code that passes a .dat file path continues to work as before.

  - Security: download URLs from the GitHub API response are validated against
    a github.com / githubusercontent.com hostname pattern before use. ProfileCode
    validation (^[\w]+$) and Test-PathInside guards from v0.53 remain in place.

  v0.56 BETA  (2026-06-14)
-------------------------------------------------------------------------------

  Two improvements to game registration accuracy:

  - Normalization now strips three additional metadata patterns used in Eggman
    dat file names: decimal version strings without a ver/v prefix (e.g. (2.10.00)),
    full ISO date strings (e.g. (2015-12-28)), and uppercase region/edition codes
    (e.g. (JPN), (USA), (EXP)). These were previously left in the normalized key,
    inflating it with garbage and reducing Dice similarity scores when comparing
    folder names against GameProfile codes. Fuzzy matching accuracy improves for
    all games with Eggman-style names.

  - Second-pass dat registration: after the main exe scan, any game folder whose
    executable name is not in any GameProfile (e.g. pcsx2x6 / ELF-based games) is
    now matched via the dat index. Exact normalized-name lookup is tried first; if
    that fails, a fuzzy scan of all dat keys is run (same auto-threshold as the
    existing fuzzy matcher). Games are then registered using the dat's ProfileCode
    and Executable path, or the first recognized file found in the folder. Before
    this fix, these folders appeared in "GAME FOLDERS NOT RECOGNISED" even when the
    dat contained their profile code.

  v0.54 BETA  (2026-06-13)
-------------------------------------------------------------------------------

  Three bug fixes identified in code review:

  - TLS 1.2 enforcement moved to script startup (was inside Invoke-ThumbnailDownload
    only). On older Windows 10 builds, PS 5.1 defaults to TLS 1.0. The ReShade
    version check (Get-ReShadeLatestVersion) and any other web request that runs
    before the thumbnail step could silently fail. Setting the protocol once at
    the top of the script guarantees all web requests use TLS 1.2 regardless of
    which feature the user runs or in what order.

  - Get-WmiObject replaced with Get-CimInstance in Invoke-GpuFixSetup.
    Get-WmiObject is deprecated (removed in PowerShell Core 6+). Get-CimInstance
    is the modern equivalent, behaves identically in PS 5.1, and does not produce
    deprecation warnings in newer environments.

  - HyperSpin 2 JSON export: wrapped $allGames in @() before ConvertTo-Json.
    In PS 5.1, ConvertTo-Json unrolls a single-element array and outputs a plain
    JSON object instead of a one-element array. HyperSpin 2 expects an array;
    a library with exactly one TeknoParrot game would produce invalid JSON and
    cause the next export to fail to parse the existing games list.

  Documentation:

  - QuickStart: added REQUIREMENTS section at the top (Windows 10+, PS 5.1,
    TeknoParrot run at least once) so users see platform requirements before
    reading through the setup steps.

  - README: added CONTENTS table at the top listing all sections, improving
    scannability for users troubleshooting a specific feature.

  v0.53 BETA  (2026-06-13)
-------------------------------------------------------------------------------

  Bug fix and new feature:

  - Bug fix: ACTION REQUIRED falsely included already-registered games.
    Games with shared executable names (e.g. game.exe) and a fuzzy similarity
    score below the auto-register threshold were shown in ACTION REQUIRED on
    every run, even when they had already been registered via TeknoParrotUI
    (a UserProfile XML existed for them). Fixed: before flagging a folder as
    needing manual registration, the script now checks whether any candidate
    profile already has a UserProfile file; if one does, it is marked "already
    set" and suppressed from ACTION REQUIRED entirely.

  - Dat file integration (optional): reads a No-Intro TeknoParrot dat file to
    auto-register games that share an executable name, without fuzzy matching.
    Set datFile in TeknoParrot-Manager.overrides.json to the full path of the
    dat file (e.g. the Eggman/RomVault collection dat). When set:
      -- Games that previously appeared in ACTION REQUIRED as manual-registration
         items are auto-registered using the dat's <GameProfile> profile code.
      -- The dat's <Executable> path (e.g. \elf\apacheM.elf) is used as the
         GamePath so the game launches from the exact correct binary.
      -- Newly registered games are shown as "Registered (dat)" in the output.
      -- The dat is loaded once before the main menu loop and reused each run.

  - Build-DatIndex function: parses a <datafile>-root XML dat (BOM-safe via
    the existing Read-Xml helper) into a normalised-name -> {ProfileCode,
    Executable} hashtable. Entries without a <GameProfile> element (e.g.
    the collection changelog and readme ZIPs) are skipped automatically.

  - Security hardening in dat integration:
      -- Profile codes from the dat are validated to contain only word
         characters [\w]+ before being used in file path construction
         (prevents path traversal via a crafted dat file).
      -- Executable paths from the dat are validated with Test-PathInside
         to confirm they stay inside the game folder before being used.
      -- A bare backslash or forward-slash in <Executable> (after TrimStart)
         produces an empty string; an explicit guard prevents this from being
         used as a path component, which would have matched the game folder
         directory rather than a file.

  v0.50 BETA  (2026-06-12)
-------------------------------------------------------------------------------

  Critical fix and three additional fixes from deep static analysis:

  - Save-Xml: File.Replace() requires the destination to already exist and
    throws FileNotFoundException for new files. Register-Games calls Save-Xml
    to CREATE new UserProfile XMLs that do not exist yet, so every first-time
    profile registration silently failed with "FAILED to register" in v0.49.
    Fixed: Save-Xml now uses File.Move for new paths and File.Replace only
    when the destination already exists (updates).

  - Restore backup integrity check: the restore function previously accepted
    any timestamped folder in FullBackup as a valid source regardless of
    whether it contained any profiles. An interrupted backup run would leave
    an empty or partial folder that, if selected for restore, would delete all
    live profiles and copy nothing back. Fixed: the selected backup must
    contain at least one XML file; restore is aborted with an error otherwise.
    The backup selection menu now shows the profile count.

  - AutoSync stale sync-state after failed folder removal: when clearing an
    existing game folder failed (files locked, access denied), the previous
    sync-state entry for that game was preserved. On the next run the ZIP
    appeared unchanged and the game was treated as up to date, permanently
    skipping re-extraction of a corrupt or partial folder. Fixed: the
    sync-state entry is now cleared on folder-removal failure so the next run
    always re-attempts extraction.

  - Thumbnail downloads: replaced WebClient.DownloadFile (no timeout, hangs
    indefinitely on a stalled connection) with Invoke-WebRequest -OutFile
    -TimeoutSec 30. 404 detection preserved via response status code.

  - README footer version tag corrected from stale v0.45 to v0.50.

  v0.49 BETA  (2026-06-12)
-------------------------------------------------------------------------------

  Security and reliability hardening (from full-script code review):

  - Added Save-Xml helper: all XmlDocument writes are now atomic (write to
    .tmp, commit via File.Replace) and BOM-less. Previously, $doc.Save() was
    called directly in four places (Register-Games, Repair-GamePaths,
    Invoke-ControlPropagation, Invoke-GpuFixSetup), each of which would
    truncate the target file to 0 bytes before writing -- leaving a corrupt
    profile on any interruption. All four sites now use Save-Xml.

  - Fixed ConvertTo-XPathStringLiteral: the concat() separator was ",'''' ,"
    (four adjacent single-quote chars in the XPath expression), which is two
    juxtaposed empty-string literals -- a syntax error in XPath 1.0. Fixed to
    use a double-quote-delimited apostrophe literal: concat('A',"'",'B').

  - Replaced [xml] cast with Read-Xml in three deploy loops (ReShade setup,
    dgVoodoo2 API detection, dgVoodoo2 deploy). The [xml] accelerator uses the
    default XmlResolver which can process external DTD/entity references;
    Read-Xml sets XmlResolver = $null (XXE prevention).

  - Crosshair index picker: added $raw.Length <= 9 guard before [int] cast at
    both the P1 and P2 prompts. Without it, a 10-digit number >= 2147483648
    would throw PSInvalidCastException and terminate the setup wizard.

  External review items addressed:

  - All Remove-Item calls already use -LiteralPath (wildcard injection is not
    possible). The Restore deletion is scoped to items inside $userProfilesDir
    via Get-ChildItem -LiteralPath. No additional path guard needed.

  - Measure-PathThroughput's File.OpenRead is already inside a try/catch that
    returns $null on any exception (lines 160-172). No change needed.

  - Expand-NumberList return value is correct: ,@($out) is the standard PS
    idiom for returning arrays without pipeline enumeration. Callers receive
    the inner array, not a 1-element wrapper.

  - UNC path depth calculation is correct: Split('\') on both base and file
    paths produces the same count of empty strings from \\, which cancel in
    the subtraction. No change needed.

  Style suggestions (comment blocks, #region markers) declined per project
  convention (minimal comments, no multi-paragraph blocks).

  v0.48 BETA  (2026-06-12)
-------------------------------------------------------------------------------

  Bug fixes:

  - Search wildcard crash: game search used PowerShell's -like operator, which
    treats square brackets as character-class wildcards. Searching for "[TP]"
    or any term containing brackets (common in romset naming conventions such
    as "Super Alpine Racer [Sega RingEdge 2] [TP]") would silently produce
    wrong or empty results instead of a literal string match. Fixed by
    switching to String.Contains(), which performs a plain substring search
    with no metacharacter interpretation.

  - Non-atomic GPU fix profile save: the XmlWriter used to write updated
    UserProfile XMLs opened the target file for writing immediately, truncating
    it to 0 bytes before any data was written. A Ctrl+C or power loss during
    the write would permanently destroy the profile. Fixed by writing to a
    .tmp file first, then committing via File.Replace() (atomic on NTFS).

  - HTML injection in crosshair preview: crosshair filenames were embedded
    unescaped into the generated HTML preview. A crafted filename containing
    HTML tags would execute in the user's browser (local file zone). Fixed by
    encoding all filename values via System.Net.WebUtility::HtmlEncode before
    inserting into the HTML.

  - Number-range overflow in game picker: Select-RegisteredGamesInteractive
    duplicated the comma/range parsing logic from Expand-NumberList without
    the overflow guards (max-digit checks for int32 overflow). Refactored to
    call Expand-NumberList, removing the duplication and inheriting its
    input validation.

  False positives from external report (not fixed):

  - [version] cast: Get-ReShadeLatestVersion already extracts only \d+\.\d+\.\d+
    via regex; $bVer is built from FileVersionInfo integer parts. Both sides of
    the comparison are pre-sanitized; no guard was missing.

  - Script truncation: the script is complete at 4531 lines. The truncation was
    an artifact of the reviewer receiving a partial copy.

  - Missing features (Restore, LaunchBox export, device survey, execution loop):
    all present in the full script.

  v0.47 BETA  (2026-06-12)
-------------------------------------------------------------------------------

  Bug fixes from v0.46 code review (Invoke-GpuFixSetup):

  - XPath injection: field names loaded from TeknoParrot GameProfile XMLs were
    interpolated directly into XPath predicates. A field name containing an
    apostrophe would crash the SelectSingleNode call with an XmlException, and
    a crafted field name could inject arbitrary XPath into the predicate. Fixed
    by adding ConvertTo-XPathStringLiteral, which wraps values in single-quoted
    literals or concat() when the value contains an apostrophe.

  - BOM written to saved UserProfile XMLs: $doc.Save(path) always prepends a
    UTF-8 BOM. TeknoParrot profiles are BOM-less. Fixed by replacing the bare
    $doc.Save() call with an XmlWriter round-trip using BOM-less UTF8Encoding,
    matching the project-wide WriteAllText convention.

  - Sort-Object AdapterRAM overflow: [uint32] cast in the GPU sort key throws
    for drivers that surface AdapterRAM as a negative (wrapped) int32 or a
    uint64 > 4 GB. The exception is silently swallowed by Sort-Object,
    producing undefined sort order and potential wrong-GPU selection. Fixed by
    casting through [double], which never overflows.

  - Silent GameProfile parse failures: empty catch {} in the dynamic field
    discovery loop dropped all XML parse errors without a log entry, making
    incomplete field discovery undetectable. Fixed by logging a WARNING.

  - Over-broad AMD Bool field match: pattern 'amd|radeon' would have added any
    Bool field whose name contained those substrings as a substring, not just
    known AMD fix fields. Tightened to word-boundary / exact-suffix patterns.

  v0.46 BETA  (2026-06-12)

  New feature: GPU compatibility fix setup (Mode 7):

  - Added Invoke-GpuFixSetup. Detects the GPU vendor (AMD / NVIDIA / Intel)

  v0.38 BETA  (2026-06-11)
-------------------------------------------------------------------------------

  Bug fixes / security:

  - Expand-ZipFileSafe: GetFullPath was called on the combined base+entry path
    before the \\?\ prefix was applied. On .NET 4.x (PS 5.1 runtime),
    GetFullPath calls the Win32 API without extended-length-path support and
    throws PathTooLongException for any path over 260 chars -- meaning the
    function failed at exactly the case it was written to solve. Fixed by
    switching to a component-level zip-slip guard (regex check for .. and
    rooted paths) that never calls GetFullPath on a long string.

  - Expand-ZipFileSafe: zip-slip guard used StartsWith without a trailing
    backslash. A ZIP entry resolving to C:\games\FooBar\evil.exe would pass a
    guard anchored on C:\games\Foo because the string is a prefix match. Fixed
    by using the component-level guard above, which is not vulnerable to this
    prefix-collision attack.

  - Expand-ZipFileSafe: removed redundant Directory.Exists check before
    CreateDirectory. CreateDirectory is idempotent and does not throw when the
    directory already exists. The check introduced a TOCTOU race window (another
    process could delete the directory between Exists and CreateDirectory).

  - AutoSync: partial extraction folder was not cleaned up when
    Expand-ZipFileSafe threw mid-extraction (e.g., disk full, I/O error). On
    the next run, the sentinel file is gone and the folder has some content, so
    the code incorrectly classified the game as already extracted and marked it
    up-to-date in the sync state -- permanently skipping re-extraction of a
    corrupt game. The inner catch now removes the partial folder before
    recording the failure.

  - All New-Item -ItemType Directory calls replaced with
    [System.IO.Directory]::CreateDirectory(). PowerShell's New-Item -Path
    interprets [ ] * ? as wildcard characters, so a path like
    E:\Games\TeknoParrot [Staged] could resolve to the wrong location or fail
    silently. The .NET method has no wildcard interpretation, is idempotent,
    and works in PS 5.1 without requiring -LiteralPath.

    etc.) are matched by folder-name similarity using a Sørensen-Dice bigram
    score. Games above the 0.72 threshold are auto-registered; games below
    appear in ACTION REQUIRED with a best-guess profile and full candidate
    list.

  - AutoSync extraction. Extracts game ZIPs from a NAS or local source into
    a local staging folder, tracking sync state to skip unchanged games.
    Interrupted extractions are detected and retried on the next run.

  - Interactive game selection. Browse A-Z, search by keyword, or extract
    all unextracted games. Paginated list with queue tracking.

  - Control propagation. Bind one game of each type; the script copies those
    controls to every other game of the same type by function match. Carries
    aim-mode settings (relative input, sensitivity, hide-cursor) between
    same-type games.

  - Device survey. Asks which controls you have and prints a tailored plan
    of which game to bind with which device.

  - Game repair. Finds broken or empty GamePaths and re-points them to the
    correct executable when the match is unambiguous.

  - LaunchBox export. Writes a LaunchBox-compatible XML file listing every
    registered game, ready for the LaunchBox import wizard.

  - Restore from backup. Lists all timestamped UserProfiles backups with file
    counts and restores the chosen one with a single YES confirmation.

  - Per-game overrides. JSON file for noSync, onlySync, noPropagate,
    forceArchetype, and familyOverride.

  - Safe by design. Full UserProfiles backup before every run, sentinel-file
    extraction tracking, free-space check, comprehensive log, [UNLOGGED]
    console echo if log is inaccessible.
