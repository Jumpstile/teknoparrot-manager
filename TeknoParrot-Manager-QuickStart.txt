  TeknoParrot Manager  |  v0.99.36 BETA  --  QUICK START
  Author: Jumpstile
===============================================================================

  Registers your games with TeknoParrot and copies controls across games of
  the same type. Full details: TeknoParrot-Manager-README.txt.

  Unfamiliar term (GamePath, UserProfile, archetype, etc.)? See the
  GLOSSARY section near the top of TeknoParrot-Manager-README.txt.

  BETA -- test one game after each run. Profiles are backed up automatically.


  NEW TO TEKNOPARROT?
  -------------------
  TeknoParrot is a free Windows frontend/loader for arcade games that
  originally ran on PC-based arcade hardware (Sega Lindbergh, Sega RingEdge,
  Taito Type X, Raw Thrills, and others). It handles per-game configuration,
  control mapping, and emulation compatibility so these games run on a
  normal PC. Official site: https://www.teknoparrot.com/

  This script is NOT TeknoParrot itself. It is a separate automation layer
  that bulk-configures TeknoParrot's per-game profiles for you instead of
  doing it one game at a time through TeknoParrotUI.

  Key terms used in this document:

    UserProfile   Your per-game settings file (game path, controls, GPU
                  flags). TeknoParrot Manager creates and updates these.

    GameProfile   TeknoParrot's own descriptor for a game -- the XML the
                  script ships for it, naming the executable and supported
                  settings. You never edit these directly.

    Dat file      A community game database (Eggman/RomVault). The script
                  uses it to tell apart games that share an executable name.

    Propagation   Copying control bindings from one game you have set up
                  to every other game of the same control type.

    Archetype     The one game per control family that you have fully bound.
                  Its bindings are the source for propagation to other games.

  For the full glossary, see the GLOSSARY section in
  TeknoParrot-Manager-README.txt.


  REQUIREMENTS
  ------------
  - Windows 10 or 11
  - PowerShell 5.1 or later (built in to Windows -- no install needed)
  - TeknoParrot installed with TeknoParrotUi.exe run at least once
    (so it has downloaded its GameProfiles folder)
  - Your games as ZIP files on a NAS/drive (for AutoSync) or already extracted


  BEFORE YOU START
  ----------------
  - Run TeknoParrotUi.exe once so it downloads its game profiles.
  - Have your games as ZIP files (for AutoSync) or already extracted.


  TYPICAL FIRST RUN
  -----------------
  1. Run TeknoParrotUi.exe once.
  2. Run TeknoParrot Manager and choose AutoSync or Register only.
  3. Launch TeknoParrotUI and verify your games appear.
  4. In TeknoParrotUI, bind one game of each controller type you use.
  5. Re-run the script -- it copies those controls to all similar games.
  6. Launch one updated game and test it.
  7. Everything else (crosshairs, ReShade, dgVoodoo2, GPU fix, FFB, BepInEx,
     Postgres -- modes 3-8 and 11) is optional and can be run any time after
     this, in any order, as many times as you like.


  RUN IT
  ------
  1. Open PowerShell in the folder where TeknoParrot-Manager.ps1 is
     (typically a Scripts subfolder inside your TeknoParrot install):

       cd "C:\path\to\TeknoParrot\Scripts"

  2. Start the script:

       .\TeknoParrot-Manager.ps1

     Blocked? Allow it for this session only:

       powershell -ExecutionPolicy Bypass -File .\TeknoParrot-Manager.ps1

  3. Pick a mode (after each mode completes, you return to this menu):
       1) AutoSync         -- extract ZIPs (e.g. from a NAS), then register.
       2) Register only    -- games are already extracted; just register.
       3) Crosshair setup  -- deploy custom cursor images to lightgun games,
                             optionally hide the Windows cursor for all guns.
       4) ReShade setup    -- add visual enhancements (optional, see below).
       5) dgVoodoo2 setup  -- fix old DX8/Glide games (optional, see below).
       6) GPU fix setup    -- apply GPU vendor fixes to registered games
                             (AMD / NVIDIA / Intel, optional, see below).
       7) FFB setup        -- wheel/stick rumble and force feedback
                             (optional, see below).
       8) BepInEx update   -- update an already-installed BepInEx to the
                             latest stable 64-bit release (optional, see below).
       9) Restore backup   -- roll UserProfiles, LaunchBox files, or Postgres
                             databases back to a previous backup.
       10) Health check    -- read-only library status, safe to run any time.
       11) Postgres setup  -- install/configure PostgreSQL for games that
                             need it (Golden Tee Live, Power Putt Live, etc).
       12) Exit            -- quit the script.

  4. Answer the folder prompts. For AutoSync, choose a LOCAL staging folder
     with free space, OUTSIDE both TeknoParrot and your ZIP source.

     Not sure yet? Answer Y to "Run in PREVIEW mode first?" to see what
     AutoSync/Register would do with zero files written. When it finishes,
     answer Y to "Apply these changes for real now?" to commit them right
     away -- no need to re-run the script from scratch.

  5. Choose what to extract (AutoSync only):

       A) All unextracted games    -- extract everything not yet on disk.
       L) Browse and select        -- paginated A-Z list, pick by number.
       S) Search by keyword        -- filter by name, pick by number.
       D) Done                     -- proceed with your selection (empty queue = skip).

     Games already in your staging folder are automatically filtered out.
     In Browse or Search, type numbers or ranges (e.g. 1,3,5-7) to add
     games to your queue.

  6. Launch TeknoParrotUi.exe -- your games now appear.


  COPY YOUR CONTROLS (OPTIONAL BUT RECOMMENDED)
  ---------------------------------------------
  1. In TeknoParrot, fully bind ONE game of each control type you use.
     Bind everything: buttons, axes, Test, Service, Coin, Start.

     Good games to bind first:
       Fighting / buttons  Street Fighter III, BlazBlue, Tekken 7
       Driving             Daytona Championship USA, Initial D, OutRun 2 SP
       Lightgun            House of the Dead 4, Aliens Extermination
       Trackball           Golden Tee Live, Silver Strike Bowling

  2. Re-run this script. After registration it runs propagation automatically,
     copying those controls to every other game of the same type.

  3. Launch ONE updated game and test it before trusting the rest.


  IMPORTANT
  ---------
  After running propagation or any optional fix (Crosshairs, ReShade,
  dgVoodoo2, GPU fix, FFB), launch one game and confirm it works before
  assuming everything succeeded.


  QUICK FIXES
  -----------
  Game won't launch          Open TeknoParrotUI, point the profile to the
                             correct .exe. Re-run the script and choose Repair.

  Game not in TeknoParrot    Check ACTION REQUIRED -- it may need manual
                             registration or needs to be extracted first.

  Extraction keeps failing   Check the log for the specific error. Verify
                             free space and that the ZIP is not corrupted.

  Controls wrong after       Restore from backup (mode 9), or delete the
  propagation                game's .xml and re-run after fixing the
                             reference game's bindings in TeknoParrotUI.

  Game won't launch (Raw     Check ACTION REQUIRED -- the script auto-detects
  Thrills, e.g. H2Overdrive, if that game's install path is too long for its
  or Yu-Gi-Oh! DT6)          engine and tells you the exact short name to
                             rename the folder to.

  Game shows a coin error    Check ACTION REQUIRED -- a few games (BlazBlue
  or similar version-        series, Tekken Tag Tournament 2) need a specific
  related failure            OLDER version of a particular file. The script
                             tells you the required CRC32 and where to get it.

  Game just won't work       Check ACTION REQUIRED -- a few specific games
  no matter what you try     are confirmed not to work on AMD or Intel GPUs.
                             If yours is listed there, it's a known
                             limitation, not something you're doing wrong.

  Wrong fuzzy match          Delete the game's .xml from UserProfiles and add
                             a forceArchetype entry in overrides.json.

  Game appears twice         Delete one of the duplicate .xml files from
  in TeknoParrotUI           UserProfiles -- keep the one with correct path
                             and any bindings you have already set.

  [UNLOGGED] on console      Log file is inaccessible -- check that the
                             TeknoParrot folder is not read-only.

  Full troubleshooting: TeknoParrot-Manager-README.txt.


  CROSSHAIR SETUP (OPTIONAL)
  --------------------------
  Mode 3 deploys custom crosshair images to all registered lightgun games
  (House of the Dead, Aliens Extermination, Terminator Salvation, etc.).

  321 crosshair designs are included in the  Crosshairs\  folder next to
  this script. You can add your own PNG images to that folder at any time
  and the script will pick them up automatically on the next run.

  When you run mode 3:
    1. A preview page opens in your browser showing all available designs.
    2. Enter the number of your chosen design for Player 1 and Player 2
       (press Enter to reuse your last choice -- no need to re-find it).
    3. The script copies those images to every registered lightgun game:
         - Standard games: P1.png + P2.png in the game's exe folder.
         - ElfLdr2 games: shared pair in the ElfLdr2 emulator folder.
         - Pcsx2x6 games: shared pair in the pcsx2x6 folder; inis\PCSX2.ini
           is also updated with cursor_path for each USB guncon2 port.
    4. You are asked whether to also hide the Windows cursor for all
       lightgun games. Answer Y to set the cursor-hide field in each
       gun game's UserProfile XML (a backup is taken automatically first).

  Run mode 3 again any time to change the crosshair design.


  RESHADE VISUAL ENHANCEMENTS (OPTIONAL)
  ----------------------------------------
  ReShade is a free tool that can make your arcade games look better on
  modern screens. It sits between the game and your screen, applying
  real-time visual improvements. Your game files are never modified.

  Common uses include sharpening, scanlines, colour enhancement, and decorative bezels.

  Your games work perfectly WITHOUT ReShade. It is 100% optional and can
  be removed at any time by deleting one DLL file from a game folder.
  No game files are ever changed.

  HOW TO SET IT UP

  If you downloaded TeknoParrot Manager as a ZIP release:
    ReShade64.dll is already in the  ReShade\  folder -- no setup needed.
    Just run the script and choose mode 4, or answer Y when prompted.

  If you cloned from GitHub (or the ReShade\ folder is empty):
    The DLLs are not in the git repository. Get them yourself:
    1. Go to  https://reshade.me  and download the installer.
    2. Run it -- when it asks which game, point it at any 64-bit
       TeknoParrot game exe. It creates a DLL in that folder.
    3. Copy that DLL into the  ReShade\  folder next to this script
       and rename it to  ReShade64.dll.  (Repeat with a 32-bit game
       exe and rename to  ReShade32.dll  if you have 32-bit games.)
    4. Run TeknoParrot Manager and choose mode 4, or answer Y when
       prompted at the end of a normal run.

  Before deploying, the script checks the Authenticode signature on your
  ReShade DLL(s) -- ReShade's own installer is code-signed. An invalid or
  missing signature is shown as a warning but does not block setup.

  Once installed, press the  Home  key while in-game to open the
  ReShade overlay and switch effects on or off.

  To remove ReShade from a game: delete the DLL (e.g. dxgi.dll,
  d3d9.dll, or opengl32.dll) from that game's folder. Nothing else
  needs to be changed.

  Full details, effect descriptions, and update instructions:
  TeknoParrot-Manager-README.txt -- RESHADE VISUAL ENHANCEMENTS section.


  DGVOODOO2 LEGACY COMPATIBILITY (OPTIONAL)
  ------------------------------------------
  Some older arcade games use DirectX 8, DirectDraw, or the Glide API from
  3dfx. On modern PCs these can cause crashes or black screens. dgVoodoo2
  translates those old calls into modern DirectX -- no game files are changed.

  Only use this for games that crash or show black screens on first launch.

  To set it up:
    1. Download the dgVoodoo2 ZIP from https://dege.freeweb.hu/dgVoodoo2/dgVoodoo2/
    2. Create a  dgVoodoo2\  folder next to this script and copy in:
         From  MS\x86\      : D3D8.dll, DDraw.dll, D3DImm.dll
         From  3Dfx\x86\   : Glide2x.dll, Glide3x.dll
         From the root     : dgVoodoo.conf
    3. Run the script and choose mode 5, or answer Y at the end of any run.
       The wizard auto-detects which games need it and shows them first.

  To remove: delete the deployed DLL file(s) from the game folder.

  Full details: TeknoParrot-Manager-README.txt -- DGVOODOO2 LEGACY
  COMPATIBILITY section.


  GPU COMPATIBILITY FIXES (OPTIONAL)
  ------------------------------------
  Many TeknoParrot games include optional fix settings for specific GPU
  vendors (AMD, NVIDIA, Intel). This step auto-detects your GPU and applies
  the right setting to every registered game that supports one.

  TeknoParrot's GameProfiles folder is scanned at runtime, so newly added
  games are always covered automatically -- no script update needed.

  To run:
    Choose mode 6 from the main menu, or answer Y when prompted at the end
    of any normal run.

  Safe to re-run any time you change your GPU or update its drivers.

  Full details: TeknoParrot-Manager-README.txt -- GPU COMPATIBILITY FIXES section.


  FORCE FEEDBACK (FFB) SETUP (OPTIONAL)
  ----------------------------------------
  Wheel/stick rumble and force feedback, covering two independent
  mechanisms -- both can be set up at once, neither requires the other:
    1) FFB Blaster -- TeknoParrot's own built-in feature. Needs a paid
       TeknoParrot membership (teknoparrot.com/en/Home/Subscription) or it
       has no effect.
    2) Third-party plugin (mightymikem/FFBArcadePlugin) -- free, no
       subscription needed, fetched live from GitHub every run. Supports
       true force feedback on FFB-capable wheels (Thrustmaster and
       similar) and rumble on Xbox/XInput-style controllers.
  If a game is covered by both, you are asked once which to use for all
  such games -- it never silently picks one for you.

  To run:
    Choose mode 7 from the main menu.

  Full details: TeknoParrot-Manager-README.txt -- FORCE FEEDBACK (FFB) SETUP section.


  BEPINEX UPDATE CHECK (OPTIONAL)
  ----------------------------------
  BepInEx is a third-party Unity plugin/modding framework some games need
  (Family Guy Bowling, Mars Sortie, NERF Arcade, Rainbow BomberGirl,
  Super Bikes 3, TMNT, and others) to get controls or fixes working.

  This ONLY updates games that already have BepInEx installed -- it never
  installs BepInEx into a game that doesn't have it. Only the latest
  STABLE 64-bit release is ever used (never 32-bit, never a pre-release).
  If anything is outdated, you're asked once to update everything at once;
  the old install is backed up first.

  To run:
    Choose mode 8 from the main menu.

  Full details: TeknoParrot-Manager-README.txt -- BEPINEX UPDATE CHECK section.


  RETROBAT / BATOCERA
  -------------------
  On first run, the script asks:

      Is this a RetroBat/Batocera installation? (Y/N)

  Answer Y and game folders are extracted as  GameName.teknoparrot  instead
  of  GameName. Registration and fuzzy matching work the same way; the suffix
  is stripped automatically before any profile comparison. The script also
  recognises folders ending in .parrot or .game if they were extracted by
  other tools. The answer is saved and never asked again.

  To switch later: delete TeknoParrot-Manager.config.json and re-run, or set "RetroBat": true
  in the JSON. To re-extract with the new naming: also delete syncstate.json.
  Full details: TeknoParrot-Manager-README.txt.


  UNATTENDED MODE
  ---------------
  Run with -Unattended to skip all prompts and let the script handle
  everything automatically -- extract new games, register, repair,
  propagate, download thumbnails, and log results:

      .\TeknoParrot-Manager.ps1 -Unattended

  Requires saved settings (run once interactively first). Designed for
  Windows Task Scheduler. Check TeknoParrot-Manager.log after each run.

  Restore mode cannot run unattended -- it needs you to pick a backup.
  The -Unattended switch always runs AutoSync or RegisterOnly depending on
  whether a ZIP source folder is configured.

  Add -DryRun to preview a scheduled run with zero files written:

      .\TeknoParrot-Manager.ps1 -Unattended -DryRun

  Full scheduling instructions: TeknoParrot-Manager-README.txt.


  THUMBNAIL DOWNLOAD (OPTIONAL)
  -----------------------------
  After registration the script offers to download game icons:

      Download thumbnails for registered games missing an icon? (Y/N)

  Answer Y and the script fetches a <ProfileCode>.png for every registered
  game that does not already have one in <TeknoParrotRoot>\Icons\ -- the
  folder TeknoParrotUI uses for game thumbnails. Icons are downloaded from:

    https://github.com/teknogods/TeknoParrotUIThumbnails

  Summary printed at the end:
    Thumbnails: 47 fetched, 12 already present, 8 not in repo.

  Not all games have a thumbnail in the repo. Missing ones are skipped
  without error and counted as "not in repo".

  CUSTOM THUMBNAILS

  You can add your own PNG icons for any game:

    1. Create a  CustomThumbnails\  folder in the same folder as this script.
    2. Name each image  ProfileCode.png  -- for example:
         Daytona3.png   HouseOfDead4.png   AliensArmageddon.png
    3. Run the script and answer Y to the thumbnail prompt.
       Your images are copied to TeknoParrot's Icons folder automatically.
       Files already in Icons are never overwritten.
       If a filename doesn't match any registered game, the script warns
       you and skips that file so nothing is silently misapplied.

  FINDING THE PROFILE CODE FOR A GAME

  Easiest way: run TeknoParrot Manager once (any mode). It writes
  TeknoParrot-Manager-controls.txt next to the script, listing every
  registered game and its exact profile code. Use those names for your
  thumbnail files. For a full explanation with examples, see the README
  under CUSTOM THUMBNAILS.


  LAUNCHBOX INTEGRATION (OPTIONAL)
  ---------------------------------
  At the end of every run the script offers to add your registered games
  straight into LaunchBox:

      Add your registered games to LaunchBox now? (Y/N)

  Answer Y and the script writes directly into LaunchBox's own library
  files (no import wizard needed). It checks LaunchBox is closed first,
  backs up the files it is about to change, and never duplicates a game
  that is already there. The first time you use this, it asks how you
  want TeknoParrot games organised:

      1) Mixed into your existing Arcade platform
      2) A separate "TeknoParrot" platform
      3) A separate platform with a name you choose
      4) Both -- Arcade AND a separate TeknoParrot platform

  Your choice is remembered for next time. New games have no box art or
  metadata yet -- in LaunchBox, right-click a game and use Edit -> Search
  to fetch it, same as for any manually-imported game.

  If anything looks wrong afterward, use menu option 9 (Restore backup) ->
  LaunchBox library backup to undo it -- this restores the exact files the
  script changed, from before it changed them.

  Prefer not to let the script touch LaunchBox directly? Answer N, then Y
  to the next question instead, to get a manual-import reference file and
  step-by-step instructions for LaunchBox's own Import wizard.


  HYPERSPIN 2 EXPORT (OPTIONAL)
  -----------------------------
  At the end of every run the script offers to add your registered games to
  HyperSpin 2's game list:

      Export registered games to HyperSpin 2? (Y/N)

  Answer Y and the script locates TeknoParrot's game list in your HyperSpin 2
  data folder (default: C:\ProgramData\HyperSpin\data) and merges in every
  registered game not already present. Your path is saved and reused on
  future runs.

  Prerequisites: TeknoParrot must already be set up as an emulator in
  HyperSpin 2 (the emulator title must contain "TeknoParrot" -- exact
  spelling and spacing do not matter). No games need to be added to
  HyperSpin 2 first; the script creates the game list file if it does
  not yet exist.

  Games are added with title only. Use HyperSpin 2's Scrape feature for
  box art and metadata. HyperSpin 2 must not be running when you answer Y.
  Full details: TeknoParrot-Manager-README.txt.


  RESTORING A BACKUP
  ------------------
  1. Close TeknoParrot completely before restoring (the script checks and
     will stop if it detects TeknoParrot is still running).
  2. Re-run the script and choose mode 9) Restore backup.
  3. The script lists all backups (most recent first) with file counts.
  4. Pick one by number, type YES to confirm.
  5. Done. Re-open TeknoParrot to use the restored profiles.


  FUZZY MATCHING  (NESiCAxLive and other shared-exe platforms)
  ------------------------------------------------------------
  Games that share an executable name (all NESiCAxLive titles use game.exe)
  are auto-registered by comparing the game folder name to every candidate
  profile code using a similarity score. Three outcomes are possible:

    Score >= 0.72   Auto-registered and shown in Cyan -- spot-check these.
    Score >= 0.40   Listed in ACTION REQUIRED with a best-guess profile shown.
                    One click in TeknoParrotUI to confirm the right one.
    Score < 0.40    Listed in ACTION REQUIRED with the full candidate list.

  Folder names are normalised before comparison: years like (2012), ISO dates
  like (2015-12-28), decimal versions like (2.10.00), region codes like (JPN),
  version strings like (ver 1.1), and bracket metadata like [NESiCAxLive] are
  all stripped. Meaningful names like (Special Edition) are kept because they
  may be the only thing distinguishing two titles.

  If a game registers to the wrong profile: delete its .xml from UserProfiles
  and add a forceArchetype entry in overrides.json to pin it on the next run.
  Full details: README section HOW FUZZY MATCHING WORKS.

  DAT FILE INTEGRATION

  During initial setup, the script asks whether you want to configure the
  Eggman/RomVault dat files:

    D) Download from GitHub now  (~145 MB, one-time)
       The script queries the GitHub API for the latest release asset, shows
       the file name and size, asks where to save it, and downloads with a
       progress bar. If GitHub is unreachable or the download fails, it falls
       through to a manual path prompt.

    Z) I have the ZIP already -- enter path
       Point to the TeknoParrot.Collection.*_RomVault.zip on disk. Both the
       collection dat and supplementary dat are read directly from inside the
       ZIP -- no extraction needed.

    F) I have separate dat files -- enter paths
       Enter the collection .dat path, then optionally the supplementary .dat.

    N) Skip (you can always re-run setup later to configure it).

  The script uses the dat's <GameProfile> codes to register three types of
  games that would otherwise appear in ACTION REQUIRED:
    - Shared-executable games (NESiCAxLive, etc.) -- dat disambiguates instantly.
    - Games with no profileIndex match (pcsx2x6, ELF-based games) -- a second
      pass looks them up by normalized folder name.
    - Slightly misnamed folders -- fuzzy scan of all dat entries.

  If you enable supplementary dat indexing, the script tracks how many
  alternate versions of your games exist (different regions, revisions, bonus
  content). Details will be shown per-game in a future release.

  Games registered via the dat are shown as "Registered (dat/exact)" or
  "Registered (dat/fuzzy)". The datFile key in overrides.json takes precedence
  over the setup-configured path. See README -- PER-GAME OVERRIDES.


  ACTION REQUIRED SUMMARY
  -----------------------
  At the end of every run the script prints an ACTION REQUIRED section
  listing everything that still needs your attention:

    Not in TeknoParrot      Folders with game files that matched no
                            TeknoParrot profile. Informational -- no
                            action needed. Shows unsupported games.

    Register these games    Games whose executable is shared by multiple
                            TeknoParrot profiles and the folder name did not
                            match confidently enough to auto-register. Shows
                            the executable, a best-guess profile (where
                            available), and the full candidate list to choose
                            from in TeknoParrotUI.

                            NESiCAxLive note: most NESiCAxLive games ARE
                            auto-registered by fuzzy matching. Only those
                            below the confidence threshold appear here, with
                            the best-guess profile shown to make picking easy.

    Fix these game paths    Profiles with a broken path that the script
                            could not auto-fix. Open TeknoParrotUI and point
                            each one to the correct game folder.

    Extract first           Games not yet in your staging folder. Extract
                            them, then re-run and choose Repair.

    Set up controls         Control types with no reference game bound yet.
                            Shows which games are waiting and suggests titles
                            to bind in TeknoParrotUI for each type.

    Setup notes             Registered games with special setup notes in the
                            community compatibility database (eggmansworld.
                            github.io/TeknoParrot). Shows the expected
                            executable name and the full notes text.

  By default the summary is saved to TeknoParrot-Manager-ActionItems.txt next
  to the script -- a Save dialog lets you pick a different location instead,
  skipped automatically during unattended runs and preview/dry-run mode.


  GOOD TO KNOW
  ------------
  - Profiles are backed up before every run:
    UserProfiles\FullBackup\<date_time>\. Nothing is ever deleted.
  - If backup folder creation fails, the script stops rather than proceeding
    without a restore point.
  - If the log file is inaccessible, a one-time warning is shown with the
    path and error reason. Every entry that cannot be archived is then echoed
    to the console prefixed with [UNLOGGED] so nothing is silently lost.
  - If an extraction is interrupted (Ctrl+C, power loss, disk error), the
    incomplete game folder is automatically detected and re-extracted on the
    next run. The staging folder is always left in a clean state.
  - Fuzzy name matching auto-registers most NESiCAxLive and other shared-exe
    games. The similarity score is shown so you can spot-check each match.
    Meaningful names like (Special Edition) are preserved during normalisation
    so they can differentiate titles with otherwise identical names.
  - Games already bound are left untouched.
  - Game-specific controls are left for you to set manually.
  - After registration, game folders with no matching TeknoParrot profile
    are listed in ACTION REQUIRED -- so you finally know which titles are
    unsupported vs just failing to register.
  - After every run, TeknoParrot-Manager-controls.txt is written next to
    the script: every game, its family, propagation source, bound count,
    and any buttons still manual. Open it when a game misbehaves weeks later.
  - On first run, the script auto-detects TeknoParrot's install path before
    prompting -- scans LaunchBox folders, drive roots, and Program Files.
  - After registering, the script offers to download missing game thumbnails
    from the TeknoParrotUIThumbnails repo into <TeknoParrotRoot>\Icons\.
    Already-present icons are never overwritten; icons not in the repo are
    skipped without error.
  - After registering, the script offers to repair any broken game paths.
  - On later runs the script remembers your settings -- press Y to reuse.
  - To fix a mis-classified game's control family (e.g. FamilyGuyBowling),
    add it to the familyOverride section of the overrides file.
  - Any time you're asked to type a file or folder path, you can type B
    instead to open a native Windows browse dialog and pick it.
